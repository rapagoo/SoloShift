SoloShift 1차 기획안

1. 프로젝트 한 줄 정의
혼자 공부하거나 취업 준비하는 사람이, 가상의 회사에 출근하듯 하루를 운영하고 기록하는 자기관리 앱.

컨셉 문장
혼자 준비해도, 일하는 사람처럼 하루를 쌓게 해주는 앱

제품 정체성
- 온라인 출근 감각 제공
- 소속감과 긴장감 보완
- 하루 작업 흐름 기록
- 가벼운 회사 롤플레잉 + 자기관리 도구

2. 해결하려는 문제
사용자가 겪는 문제
- 출근 감각이 없음
- 소속감이 없음
- 긴장감이 없음
- 하루가 흐물흐물해짐
- 실제로 시간을 썼는데 뭘 했는지 남지 않음
- 혼자 하다 보니 딴짓으로 쉽게 흐름이 깨짐

문제의 본질
이 문제는 단순한 게으름이 아니라, 외부 구조와 소속감이 사라진 상태에서 생기는 자기관리 문제다.

제품이 해결하려는 것
- 하루를 명시적으로 시작하게 만들기
- 현재 무엇을 하는지 분명하게 하기
- 하루의 흐름을 남기기
- 약한 긴장감과 보상으로 루틴을 유지하게 하기

3. 타겟 사용자
1차 타겟
- 혼자 집에서 취업 준비하는 사람
- 혼자 공부하는 졸업생/대학생
- 자격증/시험 준비생

2차 확장 타겟
- 프리랜서
- 1인 개발자
- 재택 근무자
- 콘텐츠 제작자

초기 타겟 전략
MVP 단계에서는 취준생 + 개인 학습자에 집중한다.

4. 핵심 가치 제안
이 앱은 단순한 타이머 앱이 아니다.

사용자에게 주는 가치
- 나는 오늘 출근했다는 감각
- 지금 무엇을 하는지 분명히 인식하는 경험
- 하루가 그냥 사라지지 않고 남는 느낌
- 혼자여도 흐트러지지 않게 해주는 최소 구조
- 근무하는 사람처럼 하루를 운영하는 감각

차별점
기존 타이머 앱은 몇 분 집중했는가를 측정한다.
이 앱은 출근-근무-휴식-퇴근의 흐름을 기록한다.

즉, 이 앱의 본질은 근무 감각을 만들어주는 자기관리 도구다.

5. 제품 컨셉
핵심 컨셉
가상의 회사 세계관을 활용한 자기관리 앱

방향성
- 재미를 주는 장치로 회사 세계관을 도입
- 사용자를 죄책감에 빠뜨릴 정도의 압박은 금지
- 감시 도구가 아니라 구조화 도구로 설계

제품 톤
좋은 방향
- 구조화
- 루틴 형성
- 자기 관리
- 가벼운 긴장감
- 소속감 흉내내기

피해야 할 방향
- 과도한 감시
- 수치심 유발
- 강한 처벌 중심
- 앱을 켜기 싫어지게 만드는 피드백

7. 게임화 설계 원칙
목적
- 앱을 켜고 싶게 만들기
- 출근과 퇴근 루틴에 감정적 리듬 추가
- 소속감과 긴장감 제공
- 꾸준함 유지에 도움 주기

원칙
- 도구가 중심, 게임 요소는 양념
- 처벌보다 보상 중심
- 지각/실패에도 회복 가능한 구조
- 혼내더라도 상처 주는 톤은 피하기

현재 합의된 방향
가벼운 회사 롤플레잉 + 자기관리 도구

- 소량의 포인트
- 간단한 칭찬/평가 시스템

9. 캐릭터/세계관 구조 초안
기본 구조
사용자는 가상의 회사에 출근한다.

등장 요소
- 사용자 아바타
- 팀장/부장 캐릭터
- 가상의 사무실 분위기
- 출근, 근무, 퇴근에 따른 반응

초기 캐릭터 운용 제안
초반에는 팀장 캐릭터 1명 중심으로 운영한다.

예시 반응
- 정시 출근: 좋아요. 바로 오전 업무 시작합시다.
- 15분 지각: 오늘은 조금 늦었네요. 첫 세션부터 집중해봅시다.
- 목표 달성: 오늘 흐름 괜찮았습니다. 이 정도면 합격입니다.

10. 보상 및 약한 페널티 시스템 초안
보상 예시
- 제시간 출근 시 출근 수당 포인트
- 집중 세션 완료 시 업무 성과 포인트
- 일주일 연속 퇴근 기록 시 주간 보너스
- 오늘 목표 달성 시 상사 칭찬 대사

포인트 활용(장기 확장)
- 사무실 꾸미기
- 캐릭터 코스튬
- 배경 해금

약한 페널티 원칙
초기에는 강한 손해보다 보너스 감소형 구조로 간다.

예시
- 지각 시 출근 보너스 감소
- 장시간 자리 비움 시 당일 평가 멘트 하향
- 중도 종료가 많을 경우 연속 보너스 미적용

13. 사용자 시나리오
시나리오 A: 취준생의 하루
1. 오전 10시 앱 실행
2. 팀장 캐릭터가 한 줄 인사
3. 출근 버튼 클릭
4. 오늘 목표 / 첫 작업 입력
5. 상태를 알고리즘 공부로 변경
6. 집중 세션 시작
7. 쉬는 시간에는 상태를 휴식으로 변경
8. 오후에는 포트폴리오 개발로 전환
9. 밤에 퇴근 버튼 클릭
10. 하루 요약 작성
11. 팀장 평가 멘트와 포인트 정산 확인

시나리오 B: 늦은 출근
1. 사용자가 설정한 출근 시각보다 늦게 앱 실행
2. 팀장 캐릭터가 지각 반응 표시
3. 출근 보너스 일부 감소
4. 사용자는 오늘 첫 세션부터 집중 시작
5. 퇴근 시 늦었지만 회복한 하루로 정리 가능

17. 플랫폼 선택
결론
1차는 웹 앱이 가장 적합하다.

이유
- 주 사용 환경이 책상 앞 PC/노트북
- 대시보드와 상태 표시가 웹에 잘 맞음
- 항상 켜두기 좋음
- Codex 기반 바이브 코딩으로 구현하기 수월함
- 포트폴리오로 보여주기 좋음

확장 방향
웹 MVP → 반응형 개선 → 필요 시 모바일 확장

22. 현재 시점의 최종 결론
이 프로젝트의 1차 방향은 다음과 같다.
혼자 공부하거나 취업 준비하는 사람이, 가상의 회사에 출근하듯 하루를 시작하고 기록하며 보상받는 자기관리 앱

핵심
- 게임이 본체가 아니다.
- 자기관리 도구가 본체다.
- 게임 요소는 출근 감각, 소속감, 긴장감을 보완하는 장치다.
- 초기에는 작게 시작하고, 나중에 확장한다.

현재 확정된 방향
- 웹 우선
- 개인형 중심
- 출근/상태/집중/퇴근/회고 중심
- 가벼운 회사 롤플레잉 도입
- 보상 중심, 약한 페널티
- 꾸미기/랭킹/모드 분리는 후순위
