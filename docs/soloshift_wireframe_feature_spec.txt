[프로젝트명]
SoloShift

[문서명]
와이어프레임 / 기능명세 v1

[프로젝트 한 줄 정의]
혼자 공부하거나 취업 준비하는 사람이, 가상의 회사에 출근하듯 하루를 운영하는 자기관리 앱.

========================================
1. 제품 목표
========================================

1) 출근 감각 부여
2) 현재 무엇을 하고 있는지 명확하게 기록
3) 하루의 흐름을 남김
4) 가벼운 회사 롤플레잉으로 소속감과 긴장감 제공
5) 정시 출근, 집중 세션, 퇴근 회고를 습관화

========================================
2. MVP 범위
========================================

[MVP 포함]
- 회원 식별(간단 로그인 또는 임시 로컬 사용자 구조)
- 출근 / 퇴근
- 오늘 목표 / 오늘 첫 작업 입력
- 현재 상태 변경
- 집중 세션 시작 / 종료
- 퇴근 회고 작성
- 캐릭터 반응 대사
- 포인트 계산
- 일간/주간 기록 조회

[MVP 제외]
- 친구 기능
- 랭킹
- 도전 모드
- 사무실 꾸미기
- 직급 시스템
- 동료 NPC
- 복잡한 경제 시스템
- 난이도 선택

========================================
3. 핵심 사용자 흐름
========================================

[하루 시작]
1) 앱 실행
2) 홈 화면 진입
3) 출근 전 상태 확인
4) 출근 버튼 클릭
5) 오늘 목표 / 첫 작업 입력
6) 캐릭터 반응 표시
7) 근무 시작

[근무 중]
1) 현재 상태 선택
2) 집중 세션 시작
3) 세션 종료 후 짧은 메모 남김
4) 필요 시 휴식 / 식사 / 자리비움 상태로 전환
5) 다시 업무 상태로 복귀

[하루 종료]
1) 퇴근 버튼 클릭
2) 오늘 한 일 요약 작성
3) 내일 첫 작업 작성
4) 포인트 정산
5) 캐릭터 평가 대사 출력
6) 일간 기록 저장

========================================
4. 화면 와이어프레임
========================================

----------------------------------------
4-1. 홈 화면 (메인 대시보드)
----------------------------------------

[목적]
사용자가 하루 전체 흐름을 한눈에 확인하고, 가장 자주 사용하는 기능에 즉시 접근할 수 있도록 함.

[구성 요소]
- 현재 날짜
- 현재 시간
- 사용자 이름 또는 닉네임
- 현재 출근 상태
  예: 출근 전 / 근무 중 / 휴식 중 / 퇴근 완료
- 오늘 목표 요약
- 오늘 첫 작업
- 현재 상태
- 오늘 누적 근무 시간
- 오늘 누적 집중 시간
- 현재 세션 진행 여부
- 캐릭터 프로필 이미지
- 캐릭터 한 줄 대사
- 주요 버튼
  - 출근하기
  - 상태 변경
  - 집중 세션 시작
  - 퇴근하기
  - 기록 보기

[상태별 UI]
A. 출근 전
- 출근 버튼 강조
- “오늘도 출근하시겠습니까?” 같은 대사 표시

B. 근무 중
- 현재 상태 카드 강조
- 집중 세션 버튼 강조
- 퇴근 버튼은 하단 고정

C. 퇴근 완료
- 오늘 요약 표시
- 추가 입력은 잠금
- 기록 보기 버튼 강조

----------------------------------------
4-2. 출근 시작 화면 / 모달
----------------------------------------

[목적]
하루를 명시적으로 시작하게 하고, 막연한 시작을 줄임.

[입력 항목]
- 오늘 목표
- 오늘 첫 작업
- 선택 항목: 예상 퇴근 시간

[버튼]
- 출근 시작
- 취소

[저장 데이터]
- 출근 시각
- 오늘 목표
- 오늘 첫 작업

[캐릭터 반응 예시]
- 정시 출근: “좋습니다. 바로 오전 업무 시작합시다.”
- 지각 출근: “오늘은 조금 늦었네요. 첫 세션부터 집중해봅시다.”

----------------------------------------
4-3. 상태 변경 패널
----------------------------------------

[목적]
현재 무엇을 하는지 명확히 기록.

[상태 목록 예시]
- 알고리즘 공부
- 포트폴리오 개발
- 자소서 작성
- 이력서 수정
- 휴식
- 식사
- 운동
- 자리 비움

[입력 항목]
- 상태 선택
- 선택 메모

[저장 데이터]
- 상태 종류
- 시작 시각
- 종료 시각(다음 상태로 바뀔 때 자동 기록)
- 메모

----------------------------------------
4-4. 집중 세션 화면 / 모달
----------------------------------------

[목적]
실제 작업 시간을 분리해서 기록하고 성과 포인트의 기준으로 사용.

[기능]
- 프리셋 시간 선택
  - 25분
  - 50분
  - 사용자 지정
- 시작 / 일시정지 / 종료
- 세션 종료 후 메모 입력

[입력 항목]
- 세션 길이
- 세션 종료 메모
- 완료 여부

[저장 데이터]
- 시작 시각
- 종료 시각
- 총 길이
- 완료 여부
- 연결된 상태
- 세션 메모

----------------------------------------
4-5. 퇴근 정리 화면
----------------------------------------

[목적]
하루를 마감하고, 다음날 진입 장벽을 낮춤.

[입력 항목]
- 오늘 한 일 요약
- 잘한 점
- 막힌 점
- 내일 첫 작업
- 오늘 목표 달성 여부(체크)

[출력 항목]
- 총 근무 시간
- 총 집중 시간
- 오늘 획득 포인트
- 캐릭터 평가 대사

[버튼]
- 퇴근 완료
- 취소

----------------------------------------
4-6. 기록 화면
----------------------------------------

[목적]
하루/주간 단위로 축적된 기록을 확인.

[탭 구조]
- 일간 기록
- 주간 기록

[일간 기록]
- 출근 시각
- 퇴근 시각
- 총 근무 시간
- 총 집중 시간
- 상태 로그 목록
- 오늘 회고
- 포인트 내역

[주간 기록]
- 이번 주 출근 횟수
- 평균 출근 시각
- 평균 퇴근 시각
- 총 집중 시간
- 총 포인트
- 연속 기록 일수

========================================
5. 기능 명세
========================================

----------------------------------------
5-1. 출근 기능
----------------------------------------

[설명]
사용자가 하루의 근무를 시작하는 기능.

[조건]
- 하루에 1회만 가능
- 이미 출근한 날에는 재출근 불가
- 퇴근 후 재출근 불가

[입력]
- 오늘 목표
- 오늘 첫 작업

[출력]
- 출근 시각 기록
- 현재 상태를 ‘근무 중’으로 변경
- 캐릭터 반응 출력
- 포인트 계산 가능 상태 진입

----------------------------------------
5-2. 퇴근 기능
----------------------------------------

[설명]
하루의 근무를 종료하고 회고를 남기는 기능.

[조건]
- 출근 후에만 가능
- 퇴근 후에는 당일 상태 변경 및 세션 추가 불가

[입력]
- 오늘 한 일 요약
- 내일 첫 작업
- 목표 달성 여부

[출력]
- 퇴근 시각 기록
- 일간 요약 저장
- 포인트 정산
- 캐릭터 평가 출력

----------------------------------------
5-3. 상태 변경 기능
----------------------------------------

[설명]
사용자가 현재 무엇을 하고 있는지 기록.

[조건]
- 출근 후에만 가능
- 퇴근 후 불가

[세부 규칙]
- 새로운 상태를 선택하면 이전 상태의 종료 시각 자동 기록
- 휴식 / 식사 / 자리 비움 상태도 저장되지만 집중 시간에는 포함되지 않음

----------------------------------------
5-4. 집중 세션 기능
----------------------------------------

[설명]
사용자의 실제 집중 작업 시간을 측정.

[조건]
- 출근 후에만 가능
- 퇴근 후 불가
- 업무 관련 상태에서만 활성화 가능하도록 추후 제한 검토 가능

[출력]
- 세션 길이 저장
- 완료 시 포인트 부여
- 기록 화면에서 세션 목록 확인 가능

----------------------------------------
5-5. 캐릭터 반응 기능
----------------------------------------

[설명]
가벼운 회사 세계관을 형성하는 연출 기능.

[출력 시점]
- 출근 시
- 지각 출근 시
- 목표 달성 시
- 퇴근 시

[원칙]
- 짧고 명확한 반응만 제공
- 사용자에게 상처를 주는 공격적 표현 금지
- 강한 꾸짖음보다 가벼운 긴장감 유지

----------------------------------------
5-6. 포인트 기능
----------------------------------------

[설명]
사용자의 꾸준함과 성과를 수치로 보상.

[예시 규칙]
- 정시 출근: +10
- 10분 이내 지각: +5
- 30분 이상 지각: +0
- 집중 세션 완료: +5
- 오늘 목표 달성: +15
- 퇴근 회고 작성: +5
- 연속 5일 퇴근 기록: +20

[주의]
- MVP에서는 포인트 적립과 표시까지만 구현
- 소비 시스템(꾸미기 등)은 차후 추가

========================================
6. 상태 구조
========================================

[상위 상태]
- 출근 전
- 근무 중
- 휴식 중
- 자리 비움
- 퇴근 완료

[기본 규칙]
- 출근 전: 출근만 가능
- 근무 중: 상태 변경, 세션 시작 가능
- 퇴근 완료: 모든 근무 관련 입력 종료

========================================
8. 비기능 요구사항
========================================

1) 입력이 복잡하지 않아야 함
2) 홈 화면에서 주요 기능 접근이 쉬워야 함
3) 캐릭터 연출이 앱 사용 흐름을 방해하면 안 됨
4) 하루 기록은 빠르게 저장되어야 함
5) 모바일에서도 최소 조회는 가능하게 반응형 대응 검토
